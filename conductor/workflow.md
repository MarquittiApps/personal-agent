# Workflow Context (workflow.md)

## Development Principles
1. **TDD (Test-Driven Development):** Escrever testes antes da implementação.
2. **Atomic Commits:** Um commit por unidade lógica.
3. **Conventional Commits:** Seguir o padrão `<tipo>[escopo]: <descrição>`.
4. **Surgical Changes:** Alterações mínimas necessárias para atingir o objetivo.

## Definition of Done (DoD)
- [ ] Código compilado e sem erros.
- [ ] Testes unitários passando.
- [ ] Documentação atualizada (se necessário).
- [ ] Estilo de código e linting validados.
- [ ] Requisitos de segurança (RBAC/LGPD) revisados.
