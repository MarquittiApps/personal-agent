# Specification: Dashboard de Planejamento 3-Camadas

**Track ID:** planning-dashboard_20260319
**Type:** Feature
**Created:** 2026-03-19
**Status:** Draft

## Summary

Fornecer um dashboard interativo dentro do Command Center que atue como um guia visual para o planejamento em 3 espirais/camadas (diária, semanal, mensal). Este dashboard deve ser reativo às conversas no chat, atualizando-se sempre que houver mudanças de planejamento aprovadas na sessão.

## Context

Este dashboard visa consolidar a visão de longo e curto prazo do usuário, complementando a funcionalidade de orquestração do Command Center e do Agente, permitindo iteração e controle visual em vez de apenas conversacional.

## User Story

Como **Solo-Builder**, quero **visualizar e iterar sobre meu planejamento em três horizontes (diário, semanal e mensal)** para **manter meus objetivos sempre organizados e sincronizados visualmente com as decisões tomadas via chat**.

## Acceptance Criteria

- [ ] A interface mostra 3 seções correspondentes a Diário, Semanal e Mensal.
- [ ] A IA pergunta ativamente se deve atualizar o dashboard após interações sobre planejamento no chat.
- [ ] O dashboard recebe o estado via WebSocket e recarrega seus dados em tempo real após confirmação do usuário.

## Dependencies

- Integração Omnibar & LLM Chat (EP-02) (já finalizada)
- Autenticação e conexão LangGraph (EP-01) (já finalizada)

## Out of Scope

- Gestão de equipes ou múltiplos usuários.
- Sincronização multi-dispositivo sem refresh via web (foco apenas na aba/sessão ativa do Command Center).

## Technical Notes

- É imperativo criar um listener proativo no `useSocket` e usar Zustand para a persistência em memória do lado do cliente.
- Cuidado adicional com dados de planejamento: PII e segredos empresariais não devem vazar para telemetrias ou logs expostos abertamente.
